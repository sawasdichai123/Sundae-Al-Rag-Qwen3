```

1. กดปุ่ม "Collapse" ที่ด้านล่าง Sidebar (เหนือ User Card)

## Suite 5: Bots — สร้าง / แก้ไข / ลบ Bot

### 5.1: เข้าหน้า Bots

```
ขั้นตอน:
1. คลิกเมนู "Bots" ใน Sidebar

ผลที่คาดหวัง:
- เห็นหัวข้อ "Bots"
- เห็นปุ่ม "Create bot +" สีเหลือง
- เห็นช่อง Search
- เห็น Bot cards (ถ้ามี) หรือ empty state 🤖 "ยังไม่มี Bot"
```

### 5.2: สร้าง Bot ใหม่

```
ขั้นตอน:
1. กดปุ่ม "Create bot +"
2. กรอก Name: "Test Bot E2E"
3. กรอก Description: "Bot สำหรับทดสอบ"
4. กรอก System Prompt: "คุณเป็นผู้ช่วยที่ตอบภาษาไทย"
5. เปิด Toggle "Web Enabled" (ถ้ายังปิดอยู่)
6. กดปุ่ม "สร้าง Bot" (หรือ "บันทึก")

ผลที่คาดหวัง:
- กลับมาหน้ารายการ Bot
- เห็น Bot ใหม่ "Test Bot E2E" ในรายการ
- เห็น badge "Web" สีเขียว (ถ้าเปิด web enabled)
```

### 5.3: สร้าง Bot — ไม่ใส่ชื่อ

```
ขั้นตอน:
1. กดปุ่ม "Create bot +"
2. ไม่กรอกชื่อ (ปล่อยว่าง)
3. ดูปุ่มบันทึก

ผลที่คาดหวัง:
- ปุ่ม "สร้าง Bot" ถูก disabled (กดไม่ได้)
```

### 5.4: แก้ไข Bot

```
ขั้นตอน:
1. กดเลือก Bot card ที่ต้องการแก้ไข
2. แก้ไข System Prompt เป็นข้อความใหม่
3. กดปุ่ม "บันทึก"

ผลที่คาดหวัง:
- กลับมาหน้ารายการ
- คลิกเข้า Bot เดิมอีกครั้ง → เห็น prompt ที่แก้ไขแล้ว
```

### 5.5: เชื่อม Knowledge กับ Bot

```
ขั้นตอน:
1. คลิกเข้า Bot ที่ต้องการ
2. เลื่อนลงหาส่วน "Knowledge"
3. กดปุ่ม "เลือกความรู้"
4. เลือกเอกสาร (ติ๊กเครื่องหมายถูก)
5. กดปุ่ม "เสร็จสิ้น"
6. กดปุ่ม "บันทึก"

ผลที่คาดหวัง:
- เห็นเอกสารที่เลือกแสดงเป็น chip ใต้ส่วน Knowledge
- กด × บน chip จะถอดเอกสารออก
- เอกสารที่ผูกกับ Bot อื่นแล้วจะ disabled (เลือกไม่ได้)
```

### 5.6: Toggle Web Enabled

```
ขั้นตอน:
1. คลิกเข้า Bot
2. กด Toggle "Web Enabled"
3. กดบันทึก

ผลที่คาดหวัง:

- บันทึกสำเร็จ
- กลับมาดู Bot → เห็น badge "Web" (ถ้าเปิด) หรือไม่มี (ถ้าปิด)
```

### 5.7: ลบ Bot

```

### 5.8: Search Bot

### 5.9: ปุ่ม Back

ผลที่คาดหวัง:
- ปุ่มหายไปทันที
- เห็น system message ตรงกลาง: "กำลังเรียกเจ้าหน้าที่... กรุณารอสักครู่" (ป้ายสีเหลือง)
- เห็นแถบสีฟ้า: "เจ้าหน้าที่กำลังดูแลคุณอยู่ — สามารถพิมพ์ข้อความเพิ่มเติมได้"
- ช่อง placeholder เปลี่ยนเป็น "พิมพ์ข้อความถึงเจ้าหน้าที่..."
- ช่องพิมพ์ยังใช้ได้ (ไม่ disabled)
```

### 6.9: User พิมพ์ข้อความได้หลัง Handoff

```
ขั้นตอน:
1. หลังจาก Handoff แล้ว (6.8)
2. พิมพ์ "รอเจ้าหน้าที่อยู่นะครับ"
3. กด Enter

ผลที่คาดหวัง:
- ข้อความ User แสดงฝั่งขวาตามปกติ
- Bot ไม่ตอบ (เพราะอยู่ในโหมด Handoff)
- ช่องพิมพ์ไม่ถูก lock หลังส่ง
```

### 6.10: Suggestion Cards

```
ขั้นตอน:
1. เปิดหน้า Web Chat ใหม่ (เลือก Bot อื่น หรือ refresh)
2. กดที่ Suggestion Card (เช่น "นโยบายการลาหยุด เป็นอย่างไร?")

ผลที่คาดหวัง:
- ข้อความจาก Card ถูกใส่ในช่องพิมพ์
- Focus อยู่ที่ช่องพิมพ์ พร้อมส่ง
```

## Suite 7: Inbox — ดู Session + Admin Reply + Status

### 7.1: เข้าหน้า Inbox

```
ขั้นตอน:
1. คลิกเมนู "Inbox" ใน Sidebar

ผลที่คาดหวัง:
- เห็น Layout 2 Panel:
  - ซ้าย: รายการ Session + ช่อง Search
  - ขวา: ข้อความ "เลือกเซสชันจากด้านซ้ายเพื่อดูข้อความ" (💬)
```

### 7.2: ดูรายการ Sessions

```
ขั้นตอน:
1. ดูรายการ Session ทางซ้าย (ต้องเคย Chat มาก่อน)

ผลที่คาดหวัง (ต่อ Session):
- เห็น Platform icon (💬 Web หรือ 📱 LINE)
- เห็นชื่อ User หรือ "Anonymous"
- เห็นเวลาล่าสุด (เช่น "2m" หรือ "1h")
- เห็น Status badge:
  - "Active" (สีเขียว) = AI ดูแลอยู่
  - "รับเรื่อง" (สีเหลือง) = เจ้าหน้าที่ดูแล
  - "ปิดแล้ว" (สีเทา) = จบเคส
```

### 7.3: Search Sessions

```
ขั้นตอน:
1. พิมพ์ keyword ในช่อง "ค้นหาเซสชัน..."

ผลที่คาดหวัง:
- กรอง Session ตาม keyword (ชื่อ User หรือ Platform)
```

### 7.4: คลิก Session → ดูข้อความ

```
ขั้นตอน:
1. คลิกที่ Session card ใดก็ได้

ผลที่คาดหวัง:
- Panel ขวาแสดง Chat History:
  - Header: Platform icon + ชื่อ User + Session ID (8 ตัว) + Status Badge
  - ข้อความ User (ฝั่งขวา, สีเข้ม)
  - ข้อความ Bot (ฝั่งซ้าย, avatar "S" สีเหลือง)
  - เวลาใต้ทุก bubble
- Session ที่เลือกจะ highlight ใน Panel ซ้าย (พื้นหลังอ่อนลง + เส้นขอบซ้ายสีเหลือง)
```

### 7.5: เห็น Session ที่ขอ Handoff

```
ขั้นตอน:
1. หา Session ที่มี badge "รับเรื่อง" (จากการทำ Suite 6.8)
2. คลิกเข้าไป

ผลที่คาดหวัง:
- เห็น system message "ผู้ใช้ขอพูดคุยกับเจ้าหน้าที่" (ป้ายสีเหลืองตรงกลาง)
- เห็น Action Buttons: "🤖 คืนร่างให้ AI" + "✓ ปิดเคส"
- เห็นช่อง Admin Reply Composer (ขอบสีฟ้า, placeholder "พิมพ์ข้อความตอบกลับ...")
```

### 7.6: Admin ส่งข้อความตอบกลับ

```
ขั้นตอน:
1. เลือก Session ที่มี status "รับเรื่อง"
2. พิมพ์ "สวัสดีครับ ผมมาช่วยคุณ" ในช่อง Admin Reply
3. กด Enter

ผลที่คาดหวัง:
- ข้อความ Admin ปรากฏฝั่งซ้าย:
  - Avatar "A" สีน้ำเงิน (ไม่ใช่ "S" สีเหลือง)
  - ป้าย "เจ้าหน้าที่" สีน้ำเงินเล็กๆ เหนือข้อความ
  - Bubble สีฟ้าอ่อน (blue-50) + ขอบสีฟ้า
- ช่องพิมพ์ถูกเคลียร์
```

### 7.7: Admin ส่งข้อความด้วย Shift+Enter

```
ขั้นตอน:
1. ในช่อง Admin Reply พิมพ์ "บรรทัดที่ 1"
2. กด Shift+Enter
3. พิมพ์ "บรรทัดที่ 2"

ผลที่คาดหวัง:
- ข้อความขึ้นบรรทัดใหม่ในช่อง (ไม่ส่ง)
- ข้อความแนะนำ "Enter เพื่อส่ง · Shift+Enter ขึ้นบรรทัดใหม่" แสดงอยู่
```

### 7.8: คืนร่างให้ AI

```
ขั้นตอน:
1. กดปุ่ม "🤖 คืนร่างให้ AI"

ผลที่คาดหวัง:
- Status badge เปลี่ยนเป็น "Active" (สีเขียว)
- ช่อง Admin Reply Composer หายไป
- เห็นปุ่ม "🙋 รับเรื่อง" กลับมา
```

### 7.9: ปิดเคส

```
ขั้นตอน:
1. กดปุ่ม "🙋 รับเรื่อง" เพื่อเปลี่ยนเป็น human_takeover ก่อน
2. กดปุ่ม "✓ ปิดเคส"

ผลที่คาดหวัง:
- Status badge เปลี่ยนเป็น "ปิดแล้ว" (สีเทา)
- ช่อง Admin Reply Composer ไม่แสดง
- เห็นปุ่ม "🔄 เปิดใหม่"
```

### 7.10: เปิดเคสใหม่

```
ขั้นตอน:
1. กดปุ่ม "🔄 เปิดใหม่"

ผลที่คาดหวัง:
- Status badge เปลี่ยนกลับเป็น "Active" (สีเขียว)
- เห็นปุ่ม "🙋 รับเรื่อง" แสดง
```

### 7.11: Active Session ไม่มี Admin Reply Composer

```
ขั้นตอน:
1. เลือก Session ที่มี status "Active"
2. ดูส่วนล่างของ Panel ขวา

ผลที่คาดหวัง:
- ไม่มีช่องพิมพ์ข้อความตอบกลับ
- ไม่มีปุ่มส่ง (ลูกศรสีฟ้า)
```

### 8.1: เข้าหน้า Approvals

```

### 8.2: รายการผู้ใช้รออนุมัติ

```
ขั้นตอน:
1. ดูส่วน "รออนุมัติ"

ผลที่คาดหวัง (ถ้ามีผู้ใช้รออนุมัติ):
- เห็นรายชื่อผู้ใช้ + Email + วันที่สมัคร
- เห็น avatar (ตัวย่อชื่อ พื้นหลังสีเหลือง)
- เห็นปุ่ม "อนุมัติ" สีเหลือง

ผลที่คาดหวัง (ถ้าไม่มี):
- เห็น "ไม่มีผู้ใช้ที่รออนุมัติ ✓"
```

### 8.3: อนุมัติผู้ใช้ (ถ้ามีคนรอ)

```

1. กดปุ่ม "อนุมัติ" ข้างชื่อผู้ใช้

ผลที่คาดหวัง:
- ปุ่มเปลี่ยนเป็น "กำลัง..." ชั่วครู่
- ผู้ใช้หายจากส่วน "รออนุมัติ"
- จำนวน "รออนุมัติ" ลดลง 1
- จำนวน "อนุมัติแล้ว" เพิ่มขึ้น 1
- ผู้ใช้ปรากฏในส่วน "อนุมัติแล้ว" (มี badge ✓ สีเขียว)
```

### 8.4: รายการผู้ใช้ที่อนุมัติแล้ว

```
ขั้นตอน:
1. เลื่อนลงดูส่วน "อนุมัติแล้ว"

ผลที่คาดหวัง:
- เห็นรายชื่อผู้ใช้ที่อนุมัติแล้ว (สูงสุด 20 คนล่าสุด)
- แต่ละคนมี avatar + ชื่อ + Email + Role + badge "อนุมัติแล้ว" (เขียว)
- สีจางกว่าส่วน Pending (opacity ต่ำลง)
```
