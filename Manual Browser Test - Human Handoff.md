## Test Suite H: Human Handoff — ขอเรียกเจ้าหน้าที่

### H1: ปุ่ม "ขอพูดคุยกับเจ้าหน้าที่" แสดงหลังมีข้อความ

```
ขั้นตอน:
1. Login ด้วย Admin account
2. คลิกเมนู "Web Chat" ใน Sidebar
3. เลือก Bot จาก dropdown (ถ้ายังไม่ได้เลือก)
4. สังเกตบริเวณล่างสุดของหน้า (เหนือช่องพิมพ์)

ผลที่คาดหวัง:
- ยังไม่มีข้อความ → ปุ่ม "ขอพูดคุยกับเจ้าหน้าที่" ยังไม่แสดง
```

### H2: พิมพ์ข้อความแล้วปุ่มขอเจ้าหน้าที่ปรากฏ

```
ขั้นตอน:
1. พิมพ์ข้อความในช่องแชท เช่น "สวัสดี ขอถามข้อมูลหน่อย"
2. กด Enter หรือกดปุ่มส่ง
3. รอให้ Bot ตอบกลับ (Streaming) จนเสร็จ
4. สังเกตบริเวณเหนือช่องพิมพ์

ผลที่คาดหวัง:
- Bot ตอบกลับได้ปกติ (Streaming ทีละตัวอักษร)
- เห็นปุ่ม "🙋 ขอพูดคุยกับเจ้าหน้าที่" ปรากฏเหนือช่องพิมพ์
- ปุ่มเป็นสีเหลืองอ่อน (amber) ขอบโค้งมน
```

### H3: กดปุ่มขอเรียกเจ้าหน้าที่

```
ขั้นตอน:
1. กดปุ่ม "🙋 ขอพูดคุยกับเจ้าหน้าที่"
2. สังเกตการเปลี่ยนแปลง

ผลที่คาดหวัง:
- ปุ่มเปลี่ยนเป็น "กำลังเรียกเจ้าหน้าที่..." ชั่วครู่
- เห็น system message ตรงกลาง: "กำลังเรียกเจ้าหน้าที่... กรุณารอสักครู่" (ป้ายสีเหลืองอ่อน)
- ปุ่ม "ขอพูดคุยกับเจ้าหน้าที่" หายไป (เพราะ status เปลี่ยนแล้ว)
- เห็นแถบสีฟ้าข้อความ "เจ้าหน้าที่กำลังดูแลคุณอยู่ — สามารถพิมพ์ข้อความเพิ่มเติมได้"
- ช่องพิมพ์ยังใช้ได้ (ไม่ disabled)
```

### H4: User ยังพิมพ์ข้อความได้หลัง Handoff

```
ขั้นตอน:
1. หลังจากกดขอเจ้าหน้าที่แล้ว
2. พิมพ์ข้อความเพิ่มเติม เช่น "รอเจ้าหน้าที่อยู่นะครับ"
3. กด Enter

ผลที่คาดหวัง:
- ข้อความ User แสดงฝั่งขวาตามปกติ
- Bot ยังตอบกลับได้ (streaming)
- ช่องพิมพ์ไม่ถูก lock
```

## Test Suite I: Inbox — Admin เห็นเซสชันที่ขอ Handoff

### I1: เซสชันเปลี่ยน status เป็น "รับเรื่อง"

```
ขั้นตอน:
1. คลิกเมนู "Inbox" ใน Sidebar
2. ดูรายการเซสชันทางซ้าย
3. หาเซสชันที่เพิ่งทำ Handoff (เซสชันล่าสุด)

ผลที่คาดหวัง:
- เห็นเซสชันที่ User เพิ่งขอ Handoff อยู่ด้านบนสุด (เรียงตาม last_message_at)
- status badge เป็น "รับเรื่อง" (สีเหลืองอำพัน / amber)
- เห็นชื่อ User หรือ "Anonymous"
- เห็นเวลาล่าสุด เช่น "เมื่อสักครู่"
```

### I2: เปิดเซสชันเห็น system message

```
ขั้นตอน:
1. คลิกเซสชันที่ขอ Handoff
2. สังเกตข้อความทางขวา

ผลที่คาดหวัง:
- เห็นข้อความ User (ฝั่งขวา, สีเข้ม)
- เห็นข้อความ Bot (ฝั่งซ้าย, มี avatar "S" สีเหลือง)
- เห็น system message "ผู้ใช้ขอพูดคุยกับเจ้าหน้าที่" ตรงกลาง (ป้ายสีเหลืองอ่อน)
- Header แสดง status badge "รับเรื่อง" (สีเหลืองอำพัน)
```

### I3: Admin Reply Composer ปรากฏเมื่อ status เป็น human_takeover

```
ขั้นตอน:
1. เซสชันที่เลือกต้องมี status เป็น "รับเรื่อง" (human_takeover)
2. สังเกตส่วนล่างของพื้นที่แชทขวามือ

ผลที่คาดหวัง:
- เห็นช่องพิมพ์ข้อความ "พิมพ์ข้อความตอบกลับ..."
- ช่องมีขอบสีฟ้า (blue)
- เห็นปุ่มส่งสีฟ้า (ไอคอนลูกศร)
- เห็นข้อความเล็กๆ "Enter เพื่อส่ง · Shift+Enter ขึ้นบรรทัดใหม่"
```

### I4: Admin ส่งข้อความตอบกลับ

```
ขั้นตอน:
1. พิมพ์ข้อความในช่อง Admin Reply เช่น "สวัสดีครับ ผมมาช่วยคุณ มีอะไรให้ช่วยครับ?"
2. กด Enter หรือกดปุ่มส่ง (ลูกศรสีฟ้า)

ผลที่คาดหวัง:
- ข้อความ Admin ปรากฏฝั่งซ้ายของแชท
- มี avatar "A" สีน้ำเงิน (แทน "S" สีเหลืองของ Bot)
- มีป้าย "เจ้าหน้าที่" สีน้ำเงินเล็กๆ เหนือข้อความ
- Bubble เป็นสีฟ้าอ่อน (bg-blue-50) ขอบสีฟ้า (แตกต่างจาก Bot สีเทา)
- ช่องพิมพ์ถูกเคลียร์
- เวลาแสดงถูกต้อง
```

### I5: Admin ส่งข้อความหลายข้อความ

```
ขั้นตอน:
1. พิมพ์ข้อความที่ 2 เช่น "กรุณาบอกรายละเอียดเพิ่มเติมด้วยครับ"
2. กด Enter

ผลที่คาดหวัง:
- ข้อความที่ 2 ปรากฏต่อจากข้อความแรก
- ทุกข้อความ Admin มี avatar "A" สีน้ำเงิน + ป้าย "เจ้าหน้าที่"
- ข้อความเรียงตามเวลาถูกต้อง
```

## Test Suite J: WebChat — User เห็นข้อความจาก Admin (Polling)

### J1: กลับไปหน้า Web Chat เห็นข้อความ Admin

```
ขั้นตอน:
1. คลิกเมนู "Web Chat" ใน Sidebar
2. รอ 3-5 วินาที (ระบบ poll ทุก 3 วินาที)
3. สังเกตข้อความใหม่ในแชท

ผลที่คาดหวัง:
- เห็นข้อความที่ Admin ส่งจาก Inbox ปรากฏในแชท
- ข้อความ Admin มี avatar "A" สีน้ำเงิน (ไม่ใช่ "S" ของ Bot)
- มีป้าย "เจ้าหน้าที่" สีน้ำเงินเล็กๆ เหนือข้อความ
- Bubble เป็นสีฟ้าอ่อน ขอบสีฟ้า
- แถบสีฟ้า "เจ้าหน้าที่กำลังดูแลคุณอยู่" ยังแสดงอยู่
```

**หมายเหตุ**: ถ้าเปิด WebChat ใน tab/session ใหม่จะเป็น session ใหม่ ให้ใช้ tab เดิมที่ทำ H1-H4

## Test Suite M: Message Styling — แยกแยะประเภทข้อความ

