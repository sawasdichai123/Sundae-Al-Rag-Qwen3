## Test Suite D: Knowledge Base — ดูและจัดการเอกสาร

### D1: เข้าหน้า Knowledge Base

```
ขั้นตอน:
1. Login ด้วย Admin account
2. คลิกเมนู "Knowledge Base" ใน Sidebar

ผลที่คาดหวัง:
- เห็นหน้า Knowledge พร้อมหัวข้อ "Knowledge"
- เห็นเอกสาร PDF อย่างน้อย 1 รายการ (อัปโหลดไว้แล้ว)
- เห็นช่อง Search
```

### D2: ดูรายการเอกสาร

```
ขั้นตอน:
1. ดูรายการเอกสารในหน้า Knowledge Base
2. จำชื่อเอกสารและ status ไว้

ผลที่คาดหวัง (ต่อเอกสาร):
- เห็น icon เอกสาร (ซ้าย)
- เห็นชื่อไฟล์ (ตัวหนา)
- เห็นข้อมูล mime type + ขนาด เช่น "application/pdf · 125 KB"
- เห็น status badge: "พร้อมใช้" (เขียว) หรือ "กำลังประมวลผล" (เหลือง) หรือ "ข้อผิดพลาด" (แดง)
- เห็น "Update ล่าสุด X นาทีก่อน"
```

### D3: ค้นหาเอกสาร

```
ขั้นตอน:
1. พิมพ์ชื่อเอกสาร (หรือบางส่วน) ในช่อง Search
2. ดูผลลัพธ์

ผลที่คาดหวัง:
- รายการกรองเหลือเฉพาะที่ตรงกับคำค้น
- ถ้าค้นไม่เจอ: แสดง "ไม่พบเอกสารที่ค้นหา"

ขั้นตอน (ล้างค้นหา):
1. ลบข้อความในช่อง Search ให้ว่าง
2. เอกสารทั้งหมดกลับมาแสดง
```

### D4: ลบเอกสาร (ทดสอบยกเลิกเท่านั้น)

```
ขั้นตอน:
1. Hover เมาส์บน document card
2. จะเห็นปุ่มถังขยะ (สีเทา) ปรากฏขึ้นมุมขวา
3. กดปุ่มถังขยะ
4. จะเห็นข้อความ "ยืนยันลบ" และ "ยกเลิก"
5. กด "ยกเลิก" (⚠️ อย่ากดยืนยันลบ — ต้องใช้เอกสารนี้ต่อ)

ผลที่คาดหวัง:
- เห็นปุ่ม "ยืนยันลบ" + "ยกเลิก"
- กด ยกเลิก → เอกสารยังอยู่ ไม่ถูกลบ
```

## Test Suite E: Bots — สร้าง, แก้ไข, เชื่อมความรู้

### E1: เข้าหน้า Bots

```
ขั้นตอน:
1. คลิกเมนู "Bots" ใน Sidebar

ผลที่คาดหวัง:
- เห็นหัวข้อ "Bots"
- เห็นปุ่ม "Create bot +" (สีเหลือง)
- เห็นช่อง Search
- ถ้ามี bot อยู่แล้ว → เห็น bot card(s)
- ถ้าไม่มี → เห็น empty state "ยังไม่มี Bot"
```

### E2: สร้าง Bot ใหม่

```
ขั้นตอน:
1. กดปุ่ม "Create bot +"
2. ใส่ชื่อ bot เช่น "Test Bot"
3. ใส่คำอธิบาย (optional) เช่น "บอทสำหรับทดสอบ"
4. ใส่ System Prompt เช่น "คุณคือผู้ช่วย AI ตอบคำถามเป็นภาษาไทย"
5. Toggle "เปิดใช้ Web Chat" ให้เป็น ON (สีเหลือง)
6. กดปุ่ม "สร้าง Bot"

ผลที่คาดหวัง:
- กลับไปหน้า list
- เห็น bot ใหม่ในรายการ
- bot card แสดงชื่อ + prompt/description
- มี badge "Web" (สีเขียว) ถ้าเปิด Web Chat
```

### E3: แก้ไข Bot

```
ขั้นตอน:
1. คลิกที่ bot card ที่ต้องการแก้ไข
2. เปลี่ยนชื่อ, คำอธิบาย, หรือ System Prompt
3. กด "บันทึก"

ผลที่คาดหวัง:
- กลับไปหน้า list
- ข้อมูลที่แก้ไขถูกอัพเดท
```

### E4: เชื่อมต่อ Knowledge Base กับ Bot ⭐

```
ขั้นตอน:
1. คลิกที่ bot card เพื่อเข้า Edit mode
2. เลื่อนลงไปหา section "ความรู้"
3. กดปุ่ม "เลือกความรู้" (สีเหลือง)
4. จะเห็น Modal popup "เลือกเอกสารเชื่อมต่อกับ Bot"

ผลที่คาดหวัง ใน Modal:
- เห็นรายการเอกสารทั้งหมดจาก Knowledge Base
- เอกสารที่ status "พร้อมใช้" สามารถกดเลือกได้
- เอกสารที่ "กำลังประมวลผล" หรือ "ข้อผิดพลาด" จะ disabled (จาง)
- เอกสารที่เชื่อมกับ Bot อื่นแล้ว จะเห็น "เชื่อมกับ Bot อื่น" (disabled)

ขั้นตอน (เลือกเอกสาร):
5. กดที่เอกสารที่ต้องการ → checkbox เปลี่ยนเป็นสีเหลือง ☑️
6. ด้านล่างจะเห็น "เลือกแล้ว X เอกสาร"
7. กด "เสร็จสิ้น"

ผลที่คาดหวัง หลังเลือก:
- Modal ปิด
- ใต้ปุ่ม "เลือกความรู้" เห็น chips สีเหลืออ่อนแสดงชื่อเอกสารที่เลือก
- chips มีปุ่ม ✕ สำหรับ unlink

⚠️ สำคัญ: ไม่ต้องกด "บันทึก" — การเชื่อมเอกสารจะ save ทันทีตอนกดเลือก
```

### E5: ตรวจสอบว่าเอกสารเชื่อมอยู่ (re-open)

```
ขั้นตอน:
1. กด "← Back" กลับไปหน้า Bots list
2. คลิกที่ bot เดิมอีกครั้ง
3. ดูส่วน "ความรู้"

ผลที่คาดหวัง:
- ยังเห็น chips เอกสารที่เชื่อมไว้ (ไม่หาย)
- กดปุ่ม "เลือกความรู้" → เอกสารที่เชื่อมไว้มี checkbox เหลือง ☑️

❌ ถ้าเอกสารหายไป (chips ว่าง):
→ Bug: Knowledge link ไม่ persist
→ แจ้งว่า "E5 FAIL — linked documents not showing on re-open"
```

### E6: Unlink เอกสาร

```
ขั้นตอน:
1. ในหน้า Edit Bot → ดู chips เอกสาร
2. กด ✕ บน chip ของเอกสารที่ต้องการ unlink

ผลที่คาดหวัง:
- chip หายไป
- เอกสารถูก unlink จาก bot แล้ว
```

### E7: Toggle Web Chat OFF แล้วเช็ค

```
ขั้นตอน:
1. ในหน้า Edit Bot → ปิด Toggle "เปิดใช้ Web Chat" (จากเหลือง → เทา)
2. กด "บันทึก"
3. ไปหน้า "Web Chat"
4. ดู Bot dropdown

ผลที่คาดหวัง:
- Bot ที่ปิด Web Chat จะไม่ปรากฏใน dropdown
- (หรือถ้าเลือก bot นั้นไว้ → dropdown เปลี่ยนไป bot อื่น)

ขั้นตอน (กลับ ON):
5. กลับหน้า Bots → คลิก bot → Toggle ON → บันทึก
6. กลับ Web Chat → bot กลับมาใน dropdown
```

### E8: ลบ Bot

```
ขั้นตอน:
1. หน้า Bots list → hover เมาส์บน bot card ที่ต้องการลบ
2. กดปุ่มถังขยะ (มุมขวา)
3. เห็น "ยืนยันลบ" + "ยกเลิก"
4. กด "ยืนยันลบ"

ผลที่คาดหวัง:
- bot หายจากรายการ
- ⚠️ ถ้าเป็น bot ที่ใช้ใน Web Chat → ไป Web Chat เช็คว่า dropdown เปลี่ยน

ทดสอบยกเลิก:
- กดถังขยะ → กด "ยกเลิก" → bot ยังอยู่
```

### E9: ค้นหา Bot

```
ขั้นตอน:
1. พิมพ์ชื่อ bot ในช่อง "Search Models"
2. ดูผลลัพธ์

ผลที่คาดหวัง:
- รายการกรองเหลือเฉพาะ bot ที่ตรงกับคำค้น
- ล้างช่อง search → bot ทั้งหมดกลับมา
```

## Test Suite F: Web Chat — ถามตอบกับ Bot ที่เชื่อม Knowledge

### F1: เปิด Web Chat

```
ขั้นตอน:
1. คลิกเมนู "Web Chat" ใน Sidebar
2. ดู Bot dropdown ด้านบนซ้าย

ผลที่คาดหวัง:
- เห็น bot ที่เปิด Web Chat อยู่
- เห็น "Online" badge สีเขียว
- เห็นข้อความ "Welcome to SUNDAE LLM"
- เห็นช่อง input "How can I help today?..."
```

### F2: ถามคำถามที่ตรงกับ PDF ⭐

```
ขั้นตอน:
1. เลือก Bot ที่เชื่อม Knowledge แล้ว (จาก E4)
2. พิมพ์คำถามที่เกี่ยวกับเนื้อหาใน PDF โดยตรง
   ⚠️ ต้องถามเฉพาะเจาะจง ไม่ถามกว้าง

   ✅ ตัวอย่างคำถามที่ดี:
   - "ผลกระทบทางจิตวิทยามีอะไรบ้าง" (ถ้า PDF เรื่อง psychological impact)
   - "ขั้นตอนการสมัครมีอะไรบ้าง" (ถ้า PDF เป็นคู่มือ)
   - "เงื่อนไขการรับประกันคืออะไร" (ถ้า PDF เป็นสัญญา)

   ❌ คำถามที่ไม่ดี (RAG จะตอบไม่ได้):
   - "นายมีข้อมูลอะไรบ้าง" (กว้างเกินไป)
   - "สวัสดี" (ไม่ใช่คำถาม)
   - "อะไรก็ได้" (ไม่เกี่ยวกับเอกสาร)

3. กด Enter หรือกดปุ่ม Send

ระหว่างรอ:
- เห็น Loading indicator กับข้อความเปลี่ยนตามเวลา
- ครั้งแรก: ~60-120 วินาที (โหลด model)
- ครั้งถัดไป: ~30-60 วินาที

ผลที่คาดหวัง:
- เห็น AI ตอบเป็นภาษาไทย
- คำตอบอ้างอิงจากเนื้อหา PDF จริงๆ
- ไม่ใช่ "ไม่พบข้อมูลในเอกสาร"
```

### F3: ตรวจสอบ Sources Badge ⭐

```
ขั้นตอน:
1. ดู AI bubble ที่ตอบมาจาก F2
2. มองหาส่วน "อ้างอิงจากเอกสาร" ใต้คำตอบ

ผลที่คาดหวัง:
- เห็นข้อความ "อ้างอิงจากเอกสาร" พร้อม icon
- เห็น source pills แสดง:
  - document_id (8 ตัวอักษรแรก)
  - #chunk_index
  - score เป็น % (สีเขียว)
- pills มีสีพื้นหลังเทาอ่อน

⚠️ ถ้าไม่เห็น Sources:
- อาจเป็นเพราะ reranker กรองออกหมด (score < 30%)
- ลองถามคำถามอื่นที่ตรงกับ PDF มากขึ้น
```

### F4: ถามหลายคำถามรวด

```
ขั้นตอน:
1. ถามคำถามที่ 2 (เรื่องอื่นจาก PDF)
2. รอคำตอบ
3. ถามคำถามที่ 3

ผลที่คาดหวัง:
- Chat history แสดงทุก bubble ถูกต้อง (ไม่หาย)
- User bubbles อยู่ขวา (สีเทาเข้ม)
- AI bubbles อยู่ซ้าย (สีขาว)
- เวลาแสดงถูกต้อง
- Auto-scroll ลงไปที่ข้อความล่าสุด
```

### F5: เปลี่ยน Bot ใน Dropdown

```
ขั้นตอน:
1. คลิก Bot dropdown ด้านบนซ้าย
2. เลือก Bot อื่น (ถ้ามี)

ผลที่คาดหวัง:
- Chat history ถูก clear (เริ่มใหม่)
- เห็น "Welcome to SUNDAE LLM" ใหม่
- พร้อมถามคำถามกับ Bot ใหม่
```
